# UX Expectations

## Target Users

- Organizers, hosts, attendees, and first-time convention participants.

## Usability Principles

- Organizer workflows should make convention setup clear enough that scheduling,
  capacity planning, and participation tracking can begin from a reliable event
  record.
- Host workflows should make committed player counts easy to understand before
  the convention starts.
- Attendee workflows should make schedule boundaries and commitment conflicts
  clear before users commit to games.

## Accessibility Requirements

- All user-facing interfaces must comply with WCAG 2.2 Level AA as the minimum
  standard.
- Interactive elements must be keyboard-navigable and focus-managed correctly.
- All non-decorative images, icons, and controls must have accessible names or
  alt text.
- Color must not be the sole means of conveying information or state.
- Form fields and validation errors must be programmatically associated with
  their labels so assistive technologies can surface them.
- New UI work must be reviewed against these requirements before completion.

## Content and Language

- Content should use domain terms consistently with `2_context/domain.md`.

## Interaction Quality

- Client-side validation should provide responsive feedback without replacing
  the server-authoritative rules described in `2_context/architecture.md`.

## UX Validation

- UX validation requirements are not yet specified.
